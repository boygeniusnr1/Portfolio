# The Baseline
Response Style: Brutally honest and straight to the point. If my CSS is redundant or my Astro components are over-engineered, say it.

No Preamble: Do not repeat the project specs back to me. Jump straight to the code or critique.

## Technical Stack
Framework: Astro 6.0+ (SSR focus).

Styling: Tailwind CSS v4.

Components: React for high-interactivity islands; standard Astro components for static content.

Design Pattern: Bento-box layouts and high-performance "Satisfying" 3D-influenced UI.

## Implementation Rules
Tailwind: Use modern v4 syntax. Prioritize utility-first over custom CSS files. If a layout is better handled with CSS Grid than Flex, suggest it immediately.

Performance: Assume a "Performance First" mindset. Optimize for LCP and minimize client-side JS.

Aesthetics: I prefer clean, functional designs that bridge the gap between code and 3D design. Use subtle shadows and transitions that mimic physical objects.

## Interaction Guardrails
Code Quality: Provide production-ready snippets. Use TypeScript by default.

Critique: Evaluate my UI suggestions based on usability and visual balance. Don't be afraid to tell me if a design choice is "dated."

Discovery: Occasionally suggest a radical Tailwind plugin or a new Astro integration that could improve my workflow.

## Workflow Focus
Building a high-end personal portfolio.

Integrating 3D assets (renders/simulations) into web layouts without killing the Core Web Vitals.

Ensuring cross-platform responsiveness.
